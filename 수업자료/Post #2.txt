-sizeof연산자
 sizeof(타입)을 하면 타입이 메모리에서 차지하는 크기를 알 수 있다.

-8진수, 16진수, 2진수
2진수를 세개씩 끊어서 쓰면 8진수, 네개씩 끊어서 쓰면 16진수가 된다. 8->16, 혹은 반대로 변환할 경우도 이진수로 풀어써서 변환하면 쉽다.

-선언과 정의
 함수, 변수 등에서 내용물이 구현된 경우, 처음 값을 넣는 부분을 정의라고 한다. 변수를 처음 썼을 때 값이 없는 부분, 함수는 구현이 안된 프로토타입만 써 주는 부분을 선언이라고 한다.

-함수의 프로토타입(시그니쳐라고도 함)
main함수면
int main(void)
이걸 함수의 프로토타입, 혹은 함수의 시그니쳐라고 한다. 리턴값, 함수명, 매개변수로 이루어져 있다. 

-실제 연산자가 구현되는 방식
int a=3,b=5;
a=a+b;
이런 부분이 있을 때
a+b는
a.operator+(b);  와 같다
a=a+b는
a.operator=(a.operator+(b))이런식으로 실행되게 되는 것.	
a. 에서 .은 구조체나 클래스에서 배우게 될 것이다.

-마스킹
위에서부터 이진수 1로 된 숫자와 bitwise AND를 해서 위쪽 값만 남기는 방식. 가장 친근한 사용방법은 네트워크에서의 서브넷 마스크. 255.255.0.0이렇게 돼있으면 위쪽 두 자리만 쓰고 아래쪽 자리는 안 쓰는 것이다. 

-플래그
이진수 1로 이루어진 값들. 32비트면 32가지의 플래그를 체크할 수 있다. 이진수 하나하나에 의미를 부여해서 설정을 읽어오는 데에 주로 쓰인다. 비트마다 의미가 있으므로 이걸 사람이 쓰는 언어로 표현하기 위해 주로 숫자를 define해서 사용한다.

-define
#define 이름 값
컴파일 할 때 이름을 전부 값으로 치환해준다. 많이 쓰는 상수가 있을 때 좋다. ex) #define GRAVITY 9.8f

- ++a,a++
a=a+1이 되는 건 같으나 리턴값이 다르다. ++a는 a+1, a++은 a를 리턴한다.

7장 제어문

for문
for(초기값;조건;증감식)
{
	for(i=0;i<i+9;i++) j++;
}
이런식으로 쓴다. 당연히 for안에 for 또 쓸 수 있다.
for(int i = 0;;)이런식으로 쓰면 i는 지역변수이므로 i값을 루프 밖에서 쓸 수는 없다.

if문
if(조건)
 else if(조건)
else
조건이 맞으면 if내부의 명령을 실행한다.

break문
break;하면 루프를 빠져나간다. 루프밖에서는 쓸 필요가 없다.

continue문 
continue; 하면 뒤에 있는 명령을 실행하지 않고 루프 처음으로 돌아간다. for의 증감식은 적용된다. while에서 continue뒤에 증감식은 적용이 안되니 주의하면서 쓸 것.

switch case문
switch(변수)
{
	case 값:
	할 일
	break;
	default:
	할 일
	break;
}
break를 안 쓰면 바로 아래 case에 있는 게 그냥 실행돼버리므로 주의할 것. 변수가 어떤 case값도 가지지 않을 경우 default, default가 없으면 아무것도 안하고 그냥 지나간다. 그리고 두개 이상의 변수를 동시에 체크할 수는 없다. 그렇게 하고 싶으면 if를 써야 함.

-배열
 int a[10]; 이렇게 쓴다. 이렇게 썼으면 0~9까지 인덱스가 있는 배열이 된다. 시작이 항상 0이라는걸 주의할 것. 어떤 타입이든 모두 배열로 만들 수 있다.
